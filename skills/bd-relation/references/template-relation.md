## Anti-recitation rules (for downstream skills consuming this relation)

These rules govern how puppet-ink, cast-ink, write-ink, and any skill reading this relation file must USE it:

1. **The relation file is a constraint system, not a script.** It governs behavior between these two characters — it is never narrated, quoted, or summarized in prose. A character whose relation says "never says thank you to B" does not think about not saying thank you. They just do not say it.
2. **Never use the file's vocabulary in prose.** If the file says "asymmetric power dynamic with intermittent reinforcement," the prose shows A checking B's mood before speaking, not psychological terminology.
3. **Selective activation.** A scene between A and B does not activate every section. A casual encounter activates sections 1, 4, 10. A confrontation activates 5, 7. A turning point activates 8, 9. Read the scene, identify which 2-4 sections are relevant.
4. **The characters do not know their own relation file.** They do not have insight into their dynamics, power structures, or attachment patterns. They act from these structures. If a character articulates the dynamic accurately, the writing has failed.
5. **Asymmetry is load-bearing.** If A's file says "admires B's courage" and B's file says "barely registers A," that gap IS the relation. Do not resolve it. Dramatize it.
