# Emotional Profile Template — bd-emotions

> Output template for Profile mode. Fill for each character.
> Not every field at creation — some are discovered through writing.
> The template ensures the character is considered as an emotional SYSTEM, not a collection of reactions.
> **Provenance:** Created {date} | Modified by {agent/author} | Last reviewed {date} | Snapshot: {number}

## Rules

- Every field must be CHARACTER-SPECIFIC. If the content could apply to another character, rewrite it.
- Wounds are not backstory events — they are the emotional RESIDUE. What the character concluded, not what happened.
- Defenses are what the character DOES, not what they ARE. Describe the behavior from outside.
- Triggers connect PRESENT stimuli to PAST wounds. The disproportionate response is the signal.
- Chains are habitual SEQUENCES, not single emotions. The landing state is what others perceive as the character's personality.
- The Mask is not deception — it is the version of the self the character believes they need to present. It costs energy.
- Felt / Shown / Said must diverge. If all three align, reconsider — total alignment is either extreme honesty or shallow characterization.
- **Cross-reference with bd-character Psychology is mandatory.** The emotional profile is NOT a parallel system — it is the behavioral layer of the Psychology section. Every wound in this profile must trace to a wound, lie, or vulnerability in the character's core. If a wound appears here that does not exist in Psychology, either the profile is wrong or Psychology is incomplete.
- **Cross-reference with bd-memory is expected.** Triggers in this profile should correspond to triggers in the character's memory entries. A trigger that exists here but has no supporting memory is either an unfounded claim or a gap in the memory archive.

## Anti-recitation rules (for downstream skills consuming this profile)

These rules govern how puppet-ink, cast-ink, write-ink, and any skill that reads this profile must USE it:

1. **The profile is an engine, not a script.** It drives behavior — it is never narrated, quoted, or paraphrased in prose. A character with "shame as primary defense" does not think about shame. They overcompensate, withdraw, or attack.
2. **Never use the profile's vocabulary in prose.** If the profile says "intellectualization as primary defense," the character does not "intellectualize." They explain, analyze, lecture, cite statistics — the defense manifests as behavior, not psychological terminology.
3. **Selective activation.** Not every field fires in every scene. A scene about a work conflict activates different profile fields than a scene about intimacy. Read the scene first, identify which 2-3 fields are relevant, ignore the rest.
4. **The character does not know their own profile.** They do not have insight into their wounds, defenses, or chains. They act from these structures without naming them. If a character articulates their own psychology accurately, the writing has failed.
5. **Contradictions are load-bearing.** The contradictions field exists because real people are inconsistent. Do not resolve contradictions in prose — dramatize them.
6. **Variation over repetition.** The same wound activated twice should produce DIFFERENT behavior each time — because the context, the audience, and the character's current state have changed. Same engine, different output.

## Standalone coherence declaration

This emotional profile must be usable WITHOUT reading the character's core.md. A consumer (puppet-ink, cast-ink, write-ink) reading ONLY this profile should be able to:

1. Determine the character's default emotional presentation (EMOTIONAL BASELINE)
2. Predict their reaction to a specific stimulus (TRIGGERS + CHAINS)
3. Write their emotional behavior in a scene (EXPRESSION STYLE + DEFENSE REPERTOIRE)
4. Avoid breaking the character (CONTRADICTIONS + BLIND SPOTS + FELT/SHOWN/SAID)

The cross-references to bd-character Psychology exist for AUTHORING coherence, not for downstream consumption. If a downstream skill needs to read core.md to use this profile, the profile is incomplete.

## Checklist

- [ ] Every wound traces to a specific bd-character Psychology element (wound, lie, or vulnerability)
- [ ] Every defense maps to a numbered defense mechanism in core.md
- [ ] Every trigger has a bd-memory link or an explicit "no entry yet" declaration
- [ ] Felt / Shown / Said diverge for every profiled emotion — total alignment is a red flag
- [ ] Contradictions are genuine (both sides must be simultaneously true, not alternating)
- [ ] Habitual chains have different landing states — if all chains end the same way, the character is one-dimensional
- [ ] No field could belong to a different character — specificity test passed
- [ ] The mask costs something specific — "energy" alone is too vague
- [ ] Evolution potential is honest about which wounds are load-bearing (will not heal without fundamentally changing the character)
- [ ] Delta log is append-only — no deltas were deleted, reordered, or edited after the fact
- [ ] Profile is standalone-coherent — a downstream skill can use it without reading core.md
- [ ] Anti-recitation rules are present and unmodified
- [ ] Provenance is filled (date, author/agent, snapshot number)
