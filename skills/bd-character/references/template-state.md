## Reconciliation rule

Every field in this file is a DELTA — a change from core/core.md (or from the previous snapshot if building incrementally). The following contract governs all sections:

1. **Omit unchanged aspects.** If a section has no deltas, delete the section entirely.
2. **State the delta, not the full portrait.** Write what shifted and from what. Format: "{aspect}: {was X} -> {now Y}" or "{aspect}: {new element not in core}".
3. **Psychological state is the exception.** Because psychology is systemic, the Psychological state section captures the current configuration as a present-tense snapshot — but it must explicitly reference which core elements shifted and how. Every claim must trace to a core field.
