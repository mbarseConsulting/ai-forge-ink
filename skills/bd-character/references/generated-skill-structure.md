# Generated Character Skill — Directory Structure

> This document describes the structure that bd-character generates for each character.
> Every character is a self-contained skill directory.

## Directory layout

```
{character-name}/
  SKILL.md                              # Generic orchestrator — structure + loading protocol
  SPEC.md                               # Tests only
  index.md                              # Snapshot index — which snapshot, when, what changed
  core/
    core.md                             # Permanent — identity, personality, physical baseline
    writing-rules.md                    # Permanent — always/never/traps/signals
  snapshots/
    001/
      encyclopedia/                     # Raw data — updated by bd-emotions/relations/memory
        state.md                        # Deltas only — what changed from core/ at this point
        emotional-profile.md            # Generated by bd-emotions (if invoked)
        relations/
          {other-character}.md          # Relationships at this point
        memory/
          narrative/
            {event}.md                  # What happened — accumulated
          somatic/
            {trace}.md                  # What the body registered — accumulated
      writing/                          # Compiled rules — generated by --compile from encyclopedia/
        voice.md                        # How to write dialogue (rules, not description)
        emotions.md                     # How to write emotional expression (rules, not profile)
        relations.md                    # How to write interactions per character (rules, not data)
        body.md                         # When/how to describe physicality (rules, not inventory)
    002/
      (created via --new, then edited)
    ...
  meta/
    puppets/
      notebook.md                       # Persistent positions from puppet-* sessions (NOT canon)
      _staging/                         # Satellite outputs from live sessions (review + promote)
    (future: other external-to-work sources)
```

## The three roots contract

### core/ — permanent reference

The character as forged. The exhaustive reference that never changes.

- `core.md` — Identity (birth, origin, background, identity stability, constants, detail notes), personality (wound, ghost, lie, defenses, motors, facade/core), physical baseline (appearance, voice, behavior, skills, public/private). Everything about the character in one document.
- `writing-rules.md` — Instructions for anyone writing this character. Always/never/traps/characteristic signals/common traps/deformation under altered states. Separate from core.md because it's instructions, not description.

**Rule: nothing in core/ changes because of the story.** If something changes, it's a retcon, not an evolution.

### snapshots/ — deltas in time

Numbered directories (001, 002, 003...). Each snapshot contains two subdirectories:

**encyclopedia/** — raw data, only what has changed from core/ at that specific moment.
- `state.md` — Deltas only. Physical changes, behavior shifts, voice evolution, psychological state changes (wound activation, lie status, want shift, need proximity, defense configuration, active vulnerability). Anything not in state.md is unchanged from core/.
- `emotional-profile.md` — Generated by bd-emotions if invoked.
- `relations/` — Relationships at this point. One file per significant relationship, from this character's perspective.
- `memory/` — What the character has lived. `narrative/` for events, `somatic/` for body traces.

**writing/** — compiled actionable rules, generated by `--compile` from encyclopedia/.
- `voice.md` — How to construct this character's speech.
- `emotions.md` — How to write emotional expression in charged moments.
- `relations.md` — How to write interactions with specific characters.
- `body.md` — When the body exists in prose.

### meta/ — outside the work

Content generated by sessions and tools that are NOT part of the character's canonical data.

- `puppets/notebook.md` — Persistent positions from puppet-play/puppet-cast sessions. NOT canon.
- `puppets/_staging/` — Unvalidated satellite outputs from live sessions. Must be reviewed and promoted manually.
- Future: other external sources (author notes, reference images, etc.)

### Loading logic

**Default:** Read SKILL.md frontmatter (`name`, `description`) for identification. Read `index.md` to find the current snapshot number. Load `snapshots/{current}/writing/` + `core/writing-rules.md` for writing.

**No snapshots yet:** Read `core/` only. The character hasn't evolved.

**Deep loading:** writing/ + `core/core.md` + `snapshots/{current}/encyclopedia/` (state, emotional-profile, relations, memory as needed).

**Fallback (no writing/):** Load `core/core.md` + `core/writing-rules.md` + `snapshots/{current}/encyclopedia/state.md`. Flag: writing rules not compiled.

**Specific moment:** Consult `index.md` for the snapshot number, then `core/` + `snapshots/{number}/`.

### SKILL.md — generic orchestrator

Same file for every character. Contains directory structure description and loading protocol. Only two variables in the frontmatter: `name` and `description`. Generated once at forge time, never rewritten.

### index.md — snapshot registry

```markdown
# Snapshots

| # | Label | Period | Key deltas | writing/ compiled |
|---|-------|--------|------------|-------------------|
| 001 | {human label} | {when in the story} | {top 3 changes} | {yes/no — has --compile been run for this snapshot?} |
| 002 | {label} | {when} | {top 3 changes} | {yes/no} |

Current: 001
```

- Numbered sequentially (001, 002, 003...)
- Label and period are free-form
- "Key deltas" column: top 3 changes for quick scan
- "writing/ compiled" column: tracks whether --compile has been run for this snapshot
- `Current` points to the active snapshot
- No snapshots = no index.md yet (character at baseline)

### Staleness check

When loading a character, verify that `writing/ compiled` is `yes` for the `Current` snapshot. If `no`, writing rules are stale or absent. Flag: `Run /bd-character --compile`.

## Operations

### --new (create snapshot)

1. Read index.md (or core/ if no snapshots exist yet) to find the starting point
2. If first snapshot: create `snapshots/001/encyclopedia/` with an empty state.md
3. If subsequent: copy the current snapshot directory to the next number
4. Ask the author for label, period, and what changed
5. Guide the author through editing `encyclopedia/state.md` — only the deltas
6. Update index.md (set new snapshot as Current, writing/ compiled = no)
7. Suggest `--compile` to generate writing rules for the new snapshot

### --relation (build relation)

Delegates to `/bd-relation` if available, or builds inline using `references/template-relation.md`.
Output goes to `encyclopedia/relations/{other}.md` in the current snapshot.
After creation/update: suggest `--compile` to regenerate writing/relations.md.

### --compile (generate writing rules)

Reads encyclopedia/ (core/ + current snapshot's encyclopedia/) and produces `snapshots/{current}/writing/`. Effectively mandatory — a character without writing/ is in fallback mode.

### Consumer loading

| Consumer | What to load |
|----------|-------------|
| Quick reference | SKILL.md frontmatter only |
| Scene writing (surface) | writing/ + core/writing-rules.md |
| Scene writing (deep) | writing/ + core/ + encyclopedia/ |
| Character forge/edit | core/ + snapshots/encyclopedia/ (full access) |
| Puppet incarnation | core/ + encyclopedia/ (full stack) |
